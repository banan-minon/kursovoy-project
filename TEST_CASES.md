# Документация по тестированию информационной системы "Clothing Store"

## 1. Цели тестирования
Проверка работоспособности функциональных модулей системы, верификация соответствия реализованного API требованиям технического задания, проверка безопасности доступа к данным.

## 2. Модуль автоматизированного тестирования (Unit-tests)
Для проведения тестирования использован фреймворк **Jest** и библиотека **Supertest**. Протестированы основные контроллеры: `productController`, `authController`, `orderController`.

## 3. Таблица тест-кейсов (функциональное тестирование)

| ID | Название теста | Входные данные | Ожидаемый результат | Статус |
|----|----------------|----------------|---------------------|--------|
| TC-01 | Получение каталога | GET /api/products | JSON-массив объектов товаров, статус 200 | Пройдено |
| TC-02 | Фильтрация товаров | GET /api/products?category=Shirts | Список товаров только из категории Shirts | Пройдено |
| TC-03 | Поиск товаров | GET /api/products?search=Nike | Список товаров, содержащих в названии "Nike" | Пройдено |
| TC-04 | Авторизация (негативный) | Неверный email/пароль | Сообщение "Пользователь не найден", статус 400 | Пройдено |
| TC-05 | Авторизация (позитивный) | admin@gmail.com / 123 | Получение JWT-токена и объекта user | Пройдено |

## 4. Нагрузочное тестирование
Тестирование проводилось с помощью утилиты `autocannon`.
- **Метод:** GET /api/products
- **Длительность:** 10 секунд
- **Результат:** Среднее количество запросов в секунду (Req/Sec): 480. Среднее время отклика: 124 мс.
- **Вывод:** Система стабильна под нагрузкой и соответствует нефункциональным требованиям.

## 5. Итоги тестирования
В ходе тестирования дефектов критического уровня не обнаружено. Все функциональные требования системы реализованы корректно.
